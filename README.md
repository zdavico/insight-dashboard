# InSight Survey Comparison Dashboard

An interactive React dashboard for visualizing Likert-scale survey data from Champlain College's InSight co-curricular program. Compares response distributions across cohort years (Kickoff surveys) and against Exit surveys, with histogram overlays, curve fitting (Method of Moments, Least Squares, KDE), and mean score comparisons.

Built with React + Recharts. Deployed to GitHub Pages.


## Features

- Side-by-side histogram grid for all 20 survey questions
- Cohort-by-cohort and aggregate Kickoff vs. Exit comparison modes
- Three curve fit overlays: Method of Moments, Least Squares, and KDE with adjustable bandwidth
- KDE bandwidth slider with Silverman's and Scott's rule anchor marks
- Mean score horizontal bar chart for quick cross-question comparison
- Toggle between percentage and raw count y-axes
- Bar coloring by dataset or by Likert position (diverging red-to-green)
- Multi-dataset toggle: show or hide individual cohorts independently
- Auto-detection of cohort names and question mapping from Canvas CSV exports
- Handles Canvas typos, missing sections, and multi-section files automatically

## Quick Start

### Prerequisites

- **Node.js** LTS (`node --version` to check)
- **npm** (bundled with Node)
- **Git**

### 1. Clone the project

```bash
git clone https://github.com/YOUR_USERNAME/insight-dashboard.git
cd insight-dashboard
```

Or if starting fresh:

```bash
cd ~/Desktop
npx create-react-app insight-dashboard
cd insight-dashboard
npm install recharts
```

### 2. Install dependencies

Open PyCharm's terminal (Alt+F12, or View > Tool Windows > Terminal):

```bash
npm install
```

### 3. Add project files (first-time only)

If setting up from scratch, copy these into the project:

| Source file | Destination |
|---|---|
| `insight-survey-dashboard.jsx` | `src/App.js` (rename it) |
| `build-data.js` | project root |

Delete unused defaults: `src/App.css`, `src/App.test.js`, `src/logo.svg`. If `src/index.js` imports `./App.css`, remove that line.

### 4. Add your survey data

Create the data folders if they don't exist:

```bash
mkdir Kickoff Exit
```

Drop your Canvas "Student Analysis Report" CSV exports into the matching folder. Kickoff surveys go in `Kickoff/`, Exit surveys go in `Exit/`. You can have as many CSVs per folder as you need.

### 5. Build the data

```bash
node build-data.js
```

This reads all CSVs, extracts Likert response counts by cohort, and writes `src/data.json`.

### 6. Run the dev server

```bash
npm start
```

Opens at http://localhost:3000 with hot reload.


## Using the Dashboard

### Controls

- **By Cohort / Aggregate**: View individual cohort datasets or combined Kickoff vs. Exit totals
- **Dataset buttons**: Show/hide individual datasets (multiple can be active)
- **Histograms / Mean Scores**: Switch between the 20-question histogram grid and the horizontal mean comparison chart
- **Curve Fit**: Choose MoM, LSQ, KDE, or None
- **KDE Bandwidth**: Adjust smoothing when KDE is selected. Anchor marks show Silverman's and Scott's rule values (hover for formulas, click to snap)
- **Y-Axis (% / #)**: Percentages or raw response counts
- **Bar Color (Dataset / Likert)**: Color by dataset or by Likert position

### Curve Fit Methods

| Method | What it does | Best for |
|---|---|---|
| **MoM** | Normal curve from computed mean and standard deviation | Presentations: clean values to report |
| **LSQ** | Gaussian minimizing squared error to the 5 bars | Closer visual fit than MoM |
| **KDE** | Nonparametric smooth density estimate | Seeing actual shape, including skew |

### Likert Scale Mapping

| Response | Value |
|---|---|
| Very untrue of me | -2 |
| Somewhat untrue of me | -1 |
| Neutral | 0 |
| Somewhat true of me | +1 |
| Very true of me | +2 |


## Deploying to GitHub Pages

### One-time setup

1. Create a repo at [github.com/new](https://github.com/new) named `insight-dashboard` (don't initialize with a README)

2. Install the deployment package:
   ```bash
   npm install gh-pages --save-dev
   ```

3. Edit `package.json`. Add a `homepage` field near the top:
   ```json
   "homepage": "https://YOUR_USERNAME.github.io/insight-dashboard",
   ```
   And add `predeploy` and `deploy` to the `scripts` section:
   ```json
   "predeploy": "npm run build",
   "deploy": "gh-pages -d build",
   ```

4. Push your code:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/insight-dashboard.git
   git push -u origin main
   ```

   If GitHub asks for a password, use a Personal Access Token instead (GitHub > Settings > Developer Settings > Personal Access Tokens > Tokens (classic), scope: `repo`).

5. Deploy:
   ```bash
   npm run deploy
   ```

6. In your repo on GitHub, go to Settings > Pages. Under "Source," select the `gh-pages` branch. Click Save.

7. Your site will be live at:
   ```
   https://YOUR_USERNAME.github.io/insight-dashboard/
   ```

### Updating the deployment

When you add new survey CSVs:

```bash
node build-data.js
npm run deploy
```

If you also changed code:

```bash
git add .
git commit -m "Description of changes"
git push
npm run deploy
```


## Updating Survey Data

This is the workflow you'll repeat each semester or year:

1. In Canvas, go to the Kickoff or Exit quiz
2. Export the **Student Analysis Report** (CSV)
3. Drop the CSV into the `Kickoff/` or `Exit/` folder
4. Run `node build-data.js`
5. Run `npm run deploy` to update the live site (or just `npm start` to preview locally first)

The script auto-detects cohort names from the `section` column, skips rows with no section (test accounts), and handles different Canvas question ID prefixes across quiz instances.


## How the CSV Parsing Works

Canvas exports quiz results as a CSV where each question gets two columns: a text response column (e.g., "Somewhat true of me") and a numeric score column. The question text is embedded in the header with a Canvas-specific ID prefix.

`build-data.js` handles this by:

1. Matching each of the 20 canonical questions to a CSV column using a unique keyword fragment (e.g., "identify my personal values"), independent of Canvas question IDs
2. Grouping rows by the `section` column to separate cohorts
3. Counting responses in each of the 5 Likert categories per question
4. Assigning rows with empty sections to the majority section in the file
5. Normalizing known Canvas typos (e.g., "Somwhat true of me")

The 21st question in Kickoff surveys (enrollment acknowledgement) is automatically ignored.


## PyCharm Tips

- **Terminal**: Alt+F12 (or View > Tool Windows > Terminal)
- **npm scripts**: Right-click `package.json` > Show npm Scripts, then double-click any script

## Customization

### Colors

Edit the `DATASET_COLORS` and `LIKERT_COLORS` arrays near the top of `src/App.js`.

### Chart layout

Search for `gridTemplateColumns` to change how many charts appear per row. Adjust `CHART_HEIGHT` and `CHART_MARGIN` for sizing.

### KDE default bandwidth

Search for `useState(0.7)` in the App component.

## Troubleshooting

**`command not found: npx`** -- Node.js isn't installed. Download from [nodejs.org](https://nodejs.org). Restart PyCharm after installing.

**`Module not found: Can't resolve './data.json'`** -- Run `node build-data.js` first.

**`Module not found: Can't resolve 'recharts'`** -- Run `npm install recharts`.

**CSV not showing up after build** -- Check terminal output. Common causes: wrong CSV format (must be Canvas "Student Analysis Report"), or all rows have an empty section column.

**Curves not appearing** -- Make sure a curve fit method is selected (not "None").

**GitHub Pages shows a blank page** -- Confirm the `homepage` field in `package.json` matches your GitHub username and repo name exactly. Re-run `npm run deploy`.

**`git push` rejects your password** -- GitHub requires a Personal Access Token instead of a password. Generate one at GitHub > Settings > Developer Settings > Personal Access Tokens.
