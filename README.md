# InSight Survey Comparison Dashboard

Interactive React dashboard for visualizing Likert-scale survey data from the InSight co-curricular program at Champlain College.

Compares response distributions across cohort years (Kickoff surveys) and against Exit surveys, with histogram overlays, curve fitting (Method of Moments, Least Squares, KDE), and mean score comparisons.

---

## Quick Start (First Time Setup)

### Prerequisites

- **Node.js** (LTS version): Download from [nodejs.org](https://nodejs.org) if not installed.  
  Verify with: `node --version`

### 1. Create the project

Open a terminal and navigate to where you want the project:

```bash
cd ~/Desktop
npx create-react-app insight-dashboard
cd insight-dashboard
npm install recharts
```

### 2. Add project files

Copy these files into the project:

| File | Destination |
|---|---|
| `insight-survey-dashboard.jsx` | `src/App.js` (rename it) |
| `build-data.js` | project root (`insight-dashboard/`) |
| `data.json` | `src/data.json` |

Delete the default files you don't need: `src/App.css`, `src/App.test.js`, `src/logo.svg`.

If `src/index.js` has `import './App.css'`, remove that line.

### 3. Create data folders

```bash
mkdir Kickoff
mkdir Exit
```

### 4. Add your CSV data

Drop your Canvas "Student Analysis Report" CSV exports into the appropriate folder:

- Kickoff survey exports go in `Kickoff/`
- Exit survey exports go in `Exit/`

### 5. Build the data

```bash
node build-data.js
```

This reads all CSVs, extracts the Likert response counts, and writes `src/data.json`.

### 6. Start the dashboard

```bash
npm start
```

Opens automatically at [http://localhost:3000](http://localhost:3000).

## Adding New Survey Data

This is the workflow you'll repeat each semester or year:

1. In Canvas, go to the Kickoff or Exit quiz
2. Export the **Student Analysis Report** (CSV)
3. Drop the CSV file into the `Kickoff/` or `Exit/` folder
4. In the terminal, run:
   ```bash
   node build-data.js
   ```
5. If `npm start` is running, the dashboard hot-reloads automatically. If not, run `npm start`.

That's it. The script auto-detects cohort names from the `section` column in each CSV, skips rows with no section (test accounts), and handles the different Canvas question ID prefixes across quiz instances.

You can have as many CSVs as you want in each folder. Every CSV is parsed independently, and if a single file contains multiple sections, each becomes its own dataset.

## Using the Dashboard

### Controls

- **By Cohort / Aggregate**: Toggle between viewing individual cohort datasets or combined Kickoff vs Exit totals
- **Dataset buttons**: Click to show/hide individual datasets. Multiple can be active at once
- **Histograms / Mean Scores**: Switch between the 20-question histogram grid and the horizontal mean comparison chart
- **Curve Fit (MoM / LSQ / KDE / None)**: Choose a curve overlay method. Hover over each button for a description of how it works
- **KDE Bandwidth slider**: When KDE is selected, adjust the smoothing bandwidth. Anchor marks show Silverman's and Scott's rule values (hover for formulas, click to snap)
- **Y-Axis (% / #)**: Show bar heights as percentages or raw response counts
- **Bar Color (Dataset / Likert)**: Color bars by dataset or by Likert position (diverging red-to-green)

### Curve Fit Methods

| Method | What it does | Best for |
|---|---|---|
| **MoM** (Method of Moments) | Computes μ and σ from the data, draws a normal curve | Presentations: clean μ/σ values to report |
| **LSQ** (Least Squares) | Fits a Gaussian that minimizes squared error to the 5 bars | Closer visual fit than MoM |
| **KDE** (Kernel Density) | Nonparametric smooth density estimate | Seeing the actual shape, including skew |

### Likert Scale Mapping

| Response | Numeric Value |
|---|---|
| Very untrue of me | -2 |
| Somewhat untrue of me | -1 |
| Neutral | 0 |
| Somewhat true of me | +1 |
| Very true of me | +2 |

## How the CSV Parsing Works

Canvas exports quiz results as a CSV where each question gets two columns: a text response column (e.g., "Somewhat true of me") and a numeric score column (0.0 or 1.0). The question text is embedded in the header with a Canvas-specific ID prefix like `146644530: I am able to clearly identify my personal values.`

`build-data.js` handles this by:
1. Matching each of the 20 canonical questions to a CSV column using a unique keyword fragment (e.g., "identify my personal values"). This is ID-independent, so it works across different quiz instances.
2. Grouping rows by the `section` column to separate cohorts (e.g., "InSight - Class of 2029").
3. Counting how many responses fall into each of the 5 Likert categories per question.
4. Assigning rows with empty section fields to the majority section in the file (these are typically students who dropped out or had section-swap edge cases, but their survey data is still valid for the cohort).
5. Normalizing known typos in Canvas response text (e.g., "Somwhat true of me" to "Somewhat true of me").

The 21st question in Kickoff surveys (the enrollment acknowledgement) is automatically ignored because it doesn't match any of the 20 keyword patterns.

## Deploying with GitHub Pages

This lets you share the dashboard with colleagues via a URL. No server to maintain, free hosting.

### One-time setup

#### 1. Create a GitHub account

Go to [github.com](https://github.com) and sign up if you don't have one. Free tier is all you need.

#### 2. Install Git

In PyCharm's terminal:

```bash
git --version
```

If it prints a version, you're good. If not, on Mac it will prompt you to install Xcode Command Line Tools (click Install). On Windows, download from [git-scm.com](https://git-scm.com).

#### 3. Create a repository on GitHub

Go to [github.com/new](https://github.com/new):
- Repository name: `insight-dashboard`
- Public or Private (both work with free GitHub Pages)
- Don't check any of the initialization boxes
- Click Create Repository

Keep the page open, you'll need the URL.

#### 4. Install the deployment package

In PyCharm's terminal:

```bash
npm install gh-pages --save-dev
```

#### 5. Edit package.json

Open `package.json` and make two changes.

Add a `homepage` field near the top (replace `YOUR-GITHUB-USERNAME` with your actual username):

```json
{
  "name": "insight-dashboard",
  "homepage": "https://YOUR-GITHUB-USERNAME.github.io/insight-dashboard",
  "version": "0.1.0",
```

Add `predeploy` and `deploy` to the `scripts` section:

```json
"scripts": {
  "predeploy": "npm run build",
  "deploy": "gh-pages -d build",
  "start": "react-scripts start",
  "build": "react-scripts build",
```

#### 6. Connect to GitHub and push

Run these one at a time in the terminal (replace the URL with yours from step 3):

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/insight-dashboard.git
git push -u origin main
```

If GitHub asks for a password, you need a Personal Access Token instead (GitHub no longer accepts passwords). Go to github.com > Settings > Developer Settings > Personal Access Tokens > Tokens (classic) > Generate New Token. Check the `repo` scope and use the token as the password.

#### 7. Deploy

```bash
npm run deploy
```

This builds the app and pushes it to a `gh-pages` branch. Takes about 30 seconds.

#### 8. Enable GitHub Pages

Go to your repo on GitHub > Settings > Pages (left sidebar). Under "Source" select the `gh-pages` branch. Click Save. Wait about a minute, then the dashboard is live at:

```
https://YOUR-USERNAME.github.io/insight-dashboard
```

Share that link with anyone.

### Updating the deployed dashboard

When you add new CSV data:

```bash
node build-data.js
npm run deploy
```

Two commands. The site updates in about a minute.

If you also changed code:

```bash
git add .
git commit -m "description of what changed"
git push
npm run deploy
```

## Troubleshooting

**`command not found: npx`**  
Node.js isn't installed. Download from [nodejs.org](https://nodejs.org). After installing, quit and reopen PyCharm so the terminal picks up the new PATH.

**`Module not found: Can't resolve './data.json'`**  
Run `node build-data.js` first to generate the file.

**`Module not found: Can't resolve 'recharts'`**  
Run `npm install recharts` in the project folder.

**CSV not showing up after running build-data.js**  
Check the terminal output for errors. Common issues: the file isn't a Canvas "Student Analysis Report" export (wrong format), or all rows have an empty section column.

**Curves not appearing on the histograms**  
Make sure you have a Curve Fit method selected (MoM, LSQ, or KDE). If set to "None", no curves are drawn.

**GitHub Pages shows a blank page**  
Check that the `homepage` field in `package.json` matches your actual GitHub username and repo name exactly. Re-run `npm run deploy` after fixing it.

**`git push` asks for password and rejects it**  
GitHub no longer accepts account passwords. Generate a Personal Access Token at github.com > Settings > Developer Settings > Personal Access Tokens and use that instead.
